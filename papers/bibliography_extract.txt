2.1.7 XGBoost
The ensemble method is built on the idea of training a large number of models
from which complimentary subsets might be combined to make improved, more
robust predictions than any individual model. Early work by Breiman has already
shown that averaging the predictions of many predictors learned on boostrap
samples of the data, or bagging, yields a significant reduction of variance and
improvements in generalization [36]. Soon, thereafter however, AdaBoost was
introduced by Freund and Schapire an algorithm that trains weak learners in
sequence each focusing on the examples that previous learner has mistake on so
as to transform a group of moderately accurate models into one superiorly
accurate model [37]. Leveraging these works, Friedman introduced a more
general framework called gradient boosting to solve the problem of ensemble
learning as a numerical optimization over function space [38]. Instead of
reweighting misclassified examples, for the next model trained by gradient
boosting, the fitting we are doing is to negative gradients of a loss function chosen
at previous steps, making it easily adaptable to any differentiable objective. If
decision trees are used as the base classifiers, the method is found to perform
very well in a wide range of application domains while the original definition was
not scalable for data with large number of instances and attributes.
Figure 2.1.6.1: CBAM architecture with channel and spatial attention modules [45]
XGBoost, or eXtreme gradient boosting for short, (Chen and Guestrin 2016)
is a powerful implementation of gradient Boosted Decision Trees that was released

=== PAGE 27 ===
by the same name as open-source software [12]. The system was widely
adopted, in both academia and industry with its implementation being the
prevalent algorithm at structured data notebook completions while also powering
production systems of leading technology companies. A variety of technical
advances differentiates XGBoost from previous incarnations. The first is to
expose explicit regularization in the objective function, by penalizing the number
of leaves and weight size that principle provides a general way to computationally
adjust model complexity for machine learning scope which was less available in
previous tree boosting methods [12]. The second is a weighted quantile sketch
algorithm, which makes it efficient to identify approximate splits through the
construction of split candidates that are made according to the distribution of
second-order gradient statistics and hence allows scaling for datasets with
hundreds-of-millions examples [12]. Third, we provide a sparsity-aware algorithm
which trains the default direction of missing values at each split to handle sparse
data efficiently without pre-processing inspection [12]. In addition to these
algorithmic contributions, XGBoost implemented systems-level optimizations such
as the column-block data structure for parallel computing of split candidates, a
cache-aware prefetching method to reduce memory access latency, and support
for out-of-core learning when the training dataset does not fit in memory [12].
In the transportation-and-urban-mobility industry, gradient boosting
methods have been widely used for demand prediction. Ergöl Aydın et al. (2023)
compared three gradient boosting algorithms to forecast demand of bike-sharing
systems and found that XGBoost, LightGBM, and CatBoost performed well on the
task; the temporal feature hour of day and environmental bi-variate relationships
temperature were ranked among the top influencing predictors for all variants [39].
Their results showed that the complex and non-linear interplay among weather,
time of day (TOD) and ridership patterns which characterize demand could be
effectively learnt by means of tree-based ensemble methods. The integration of
gradient boosting with deep learning has continued to gain traction as well, such
as hybrid models using the temporal learning capabilities of a neural network
coupled with XGBoost’s ability to model non-linear feature interactions. In these
two-stage methods, a recurrent or convolutional network extracts pre-trained
features from the sequential input, and these features are later concatenated with

=== PAGE 28 ===
original ones to an XGBoost regressor for making prediction. This approach builds
on the strengths of both paradigms: deep learning is applied to model temporal
dynamics that tree-based methods cannot easily capture, whereas XGBoost
learns tabular feature interactions (categorical or one-hot- encoded), and
provides built-in regularization as well as interpretable feature importance scores
indicating which input variables are contributing most to predictions [12].
2.2 Feature Engineering for Time Series
Forecasting performance is a joint function of the architecture and quality of the
input features to the model. For bike-sharing demand quantity prediction, using
the raw hourly counts and meteorological measurements directly cannot fully
reflect the complicated patterns beneath because multiple factors are actually
affecting ridership through interaction and they distribute on a different range of
time scale. Feature engineering deals with this via the transformation of raw data
into a more informative set of input variables, which make these patterns more
readily apparent to the learning algorithm [7].
The simplest is generating lag feature, which will allow the model to access
previous values of target variable. There are strong temporal dependencies in the
hourly demand, where 1 hour lag could capture short-term momentum, a 24 hour
lag would contain daily commuting cycle and a 168 hours lag represents weekly
periodicity [9]. By incorporating lag horizons at multiple time scales, the model can
have simultaneous access to both the immediate past, as well as what happened
n hours ago on the same day and one week ago at n-th hour of a day — which is
essential to capture both regular commuting patterns (due to people’s daily
routine) and event driven variation. Rolling window statistics act as a complement
to lag features by taking aggregate measures like the average, standard deviation
over a sliding window of recent observations, smoothing short-term noise and
capturing local trends in demand [10].
Temporal features like hour of the day and day of week contain valuable
information on what to expect in demand levels, but treating them as plain
integers is not representative of their cyclical nature. Hour 23 is “next to” hour 0,
in the sense of being adjacent in time but maximally distant as integers, and

=== PAGE 29 ===
likewise for Sunday and Monday. Cyclical encoding addresses this by representing
each temporal variable using sine and cosine components which maintain circular
continuity while allowing the model to learn smooth periodic patterns that spans
cycle boundaries [10].
Figure 2.2.1: Lag features and cyclical time encoding for hourly time-series modelling [47]
The temporal behaviour of weather variables often interacts in a non-linear
way, which is not reflected by raw features. Temperature has an inverted-U
relationship with ridership, for example, and a warm morning rush hour creates far
different demand than a rainy morning during which it is equally warm. Using the
same Seoul Bike Sharing dataset used in this thesis, Sathishkumar and Cho
(2020) explicitly found through their variable importance analysis that temperature
and hour of the day are the top two important predictors for hourly demand [7].
They can add dummy variables to represent these so that the model has an easy
way to incorporate this conditional information. Binary features for holiday,
weekend and system operational status further contribute to context not inferred
from the continuous features. Combined, these methods make up the feature
engineering pipeline used in this thesis where the raw data is transformed into a
high-dimensional input space for training and testing the demand forecasting
model.

2.3 Bike-Sharing Systems and Demand Forecasting
Bike-sharing programs have developed over the decades, through a series of
distinct generations, beginning with the experimental deployment of free bikes in
Amsterdam in 1965 (a program that quickly folded because the majority of its bikes
were stolen or vandalized). In the 1990s coin-deposited systems was first
introduced in Copenhagen and early 2000, smart card access, GPS tracking and
automatic stations were introduced as third generation docked system [1]. The
current generation is app-based, the apps serve to find, unlock, and provide
payment for the bikes: scaling city per city. As of the end of 2020, more than 2,000
bicycle-sharing systems were deployed worldwide, which has a scale of about 1.6
billion USD and is still growing through the current decade, as per by [1]. These
have been increasingly incorporated into urban mobility plans, providing a low
emission last mile solution in combination with high capacity public transport.
Figure 2.3.1: Seoul map with bike stations [48]

=== PAGE 31 ===
Public bike-sharing system in Seoul, Ddareungi was introduced along the
Han River in October 2015 with approximately 150 stations and 1,500 bikes [3].
The system had grown rapidly, with more than 20,000 bikes at over 1,500 rental
stations available in an instant through mobile apps [3]. According to the Seoul
Metropolitan Government Usage data, 54 percent of daily trips are rush-hours (in
the mornings and evenings) and that almost 60 percent of such trips go less than
four kilometers [4]. This scale, wide variety of usage patterns and rich hourly
information make the Seoul system an interesting case for demand forecasting
research.
The market demand for shared bicycles is determined by multiple types of
factors. Weather has a considerable impact, with mild temperatures having a
positive effect on cycling, and rain, snow, cold and heat having the opposite effect.
Sathishkumar, Cho (2020) found that the temperature and hour of day are among
the first two most important input factors contributing to hourly bike rentals in
Seoul scheme [7]. Temporal dimension: the demand during weekdays is
characterized by bimodal commuting peaks in the morning and evening, while
holidays or weekends generate flatter profiles compared to weekdays, because
passengers normally use public transport for leisure purposes. These effects are
magnified over seasonal variations where winter months see far less ridership
than summer. Those factors don't operate in isolation, but bounce off each other
and combine together in complex, non-linear ways — a warm day with the sun
overhead on a Thursday morning during rush hour will produce very different
demand levels than that exact same warm weather combined with rain clouds on
a holiday afternoon — which makes it difficult to forecast accurately for any
individual modeling effort.
The central operational value of demand prediction lies in repositioning, a
challenge common to all station-based bike-sharing systems. Since users take
and return bikes at different places, stations gradually drift to empty or full, which
both compromise user experience and reduce system usage. To maintain
operation, operators must redistribute bicycles in the network and this is achieved
efficiently based on good and reliable short-term predictions where demand will
come up [1]. Miscalculated predictions result in empty stations that irritate riders
and overflowing stations where returns can’t be made, increased cost from

=== PAGE 32 ===
reactive operations vs proactive rebalancing. With today’s systems growing
rapidly, the separation between responsive management and prediction-fuelled
operations is growing larger, making accurate demand forecasting more critical to
a sustainable system.
2.4 Related Work
In the following sections, we will discuss relevant related works for demand
forecasting in bike-sharing and other transportation systems. The review is
organized based on the methodological paradigm, covering from conventional
statistical methods to machine learning and deep learning approaches and
towards the recent hybrid architectures and attention-based models. For each
category, exemplary studies are presented with focus on the used data sets,
forecasting setting and their results. This framework is representative for the whole
research field and is used to position the current work with respect to existing
research, and also to identify what exact gaps this thesis will try to fill.
2.4.1 Traditional and Statistical Approaches
The first generation of bike-sharing demand prediction studies mostly leveraged
classical time series operation models and regression approaches best practices
in common transportation planning activities. Autoregressive integrated moving
average (ARIMA) models were one of the earliest techniques used, given that they
offer a established framework for modeling univariate time series with linear
temporal dependencies through autoregressive, differencing and moving average
components [11]. Although ARIMA and its seasonal extension SARIMA can
provide decent short-term forecasting results when the signal process is rather
linear trend-stationary, they fail to capture the non-linear relationships between
weather, time and demand as they seem inherent to bike-sharing usage patterns.
Regression models Generalizations of the previous framework were
suggested by directly incorporating external variables like temperature, humidity,
wind speed and calendar effects in addition to the temporal dimension of the
series. Sathishkumar and Cho (2020) utilized numerous rule-based and

=== PAGE 33 ===
regression models on the Seoul Bike Sharing dataset that is used in this thesis,
their most successful model CUBIST obtained ar R-squared of around 95% under
the test data [7]. This result set a very high bar for the dataset, and also showed
that judiciously-designed statistical models including weather as an attribute can
account for most of the variability in hourly demand. However, the evaluation
protocol in that work employed same-time prediction, in which the model was
allowed to use current hour’s weather observations when predicting current hour
demand. In an operational scenario, predictions need to be made based on
conditions yet to be observed, which makes this a stronger and harder prediction
task. Min and Jung (2021) compared vector autoregression with machine learning
and deep learning baselines on daily bike-sharing data from 389 stations in Seoul;
while the VAR model could capture overall temporal trends, it was outperformed
across their comparison by non-linear methods [25].
These studies demonstrate the clear picture of conventional methods.
Statistical models have the advantages of interpretability, strong theoretical
properties and simple computational need, which are used as baseline model for
demand forecasting. Meanwhile, the linearity assumption and their limited ability
to capture complex interaction effects of covariates impede their performance in
situations where demand is influenced by weather condition, day-of-week/season
cycles as well as contextual features collectively. As a consequence of such
limitation, the focus has moved towards machine learning and in later days deep
learning approaches that are described in the subsequent sections.
2.4.2 Machine Learning Approaches
Pitfalls of linear statistical models capturing complex interactions among weather,
temporal and ridership fed the research to implement machine learning techniques
for forecasting bike-demand. Compared to regression-based methods, which
require the practitioner to specify the functional form of input-output relationship,
machine learning algorithms like random forest and gradient boosting are
inherently capable of learning non-linear mappings from data and are well suited
to the multi-factorial aspect of demand prediction.
Aghe (2023) evaluated a variety of standard machine learning regressors
on the Seoul Bike Sharing dataset, which included linear regression, decision tree

=== PAGE 34 ===
regression, random forest, bagging regressor and stacking regressor from
combining multiple base learners [40]. The one with the best performance on test
set was stack ensemble (R-squared : 89.56 percent), which means meta-learning
to combine models in different ways can increase generalization than using just a
single algorithm. Ergöl Aydın et al. (2023): Also did a specific comparison of
gradient boosting algorithms for bike-sharing demand: They compared XGBoost,
LightGBM and CatBoost on real-world datasets from Konya and Washington D.C.
[39]. All the three algorithms reached strong prediction accuracy and CatBoost
demonstrated slightly better generalization performance after hyperparameter
tuning. Common predictors between both studies included temperature (°F) and
time of day and confirmed the importance of weather-temporal effects as a primary
driver behind energy demand.
These machine learning techniques have practical advantages.
Ensembles of decision trees are used in practice on account of their ability to
naturally accommodate mixed types of features, and they have become popular
because they minimize the amount of preprocessing needed and provide
automatic importance scores that give insight into which variable is inducing
predictions. These methods process each individual data point independently and
do not capture the notion of temporal dependency between subsequent time
steps. A prediction for a time point such as 8 AM on a day is issued in isolation and
without structural knowledge of what actually happened at 7 AM or what usually
occurs at 8 AM on the same type of days, unless these orderings are explicitly
included via feature engineering. Due to their dependence on hand-engineered
temporal features, the amount of sequential structure that these models can
exploit is a priori bounded and it is precisely this bound that deep learning methods
presented in the next section try to overcome, by allowing the model itself to learn
suitable temporal representations directly from raw sequence.
2.4.3 Deep Learning Approaches
Deep learning approaches overcame the limitation of sequential modeling in
machine learning by automatically extracting temporal representations from raw
sequences without the need for engineered lag and window properties. Min and

=== PAGE 35 ===
Jung (2021) also evaluated a vector autoregression model, support vector
regression model, and LSTM for public bicycle demand in Seoul with the daily data
of 389 stations. Their results showed that the best performing model was an
LSTM that has high complexity than other models introduced by them, supporting
our argument of recurrent architecture being more apt at catching temporal
domain dependencies in ridership data [25]. LSTM networks, for the prediction of
station-free bike-sharing demand in Nanjing and obtained at least acceptable
accuracy in periods from ten to thirty minutes indicating that deep learning
systems were capable of processing more fine-grained temporal patterns
necessary for operational planning.
The implementation of (de)convolutional architectures has also been
investigated. Lin et al. (2018) proposed a graph convolutional neural network for
station level hourly demand prediction in a large-scale bike-sharing network,
taking spatial relationships between stations into consideration by capturing the
geographic dependence ignored by purely temporal models [33]. Li et al. (2023)
presented an irregular convolutional neural network coupled with LSTM for short-
term bike-sharing demand forecasting, where geographically distant stations can
have similar usage patterns because of the similarities in their functionality [41].
Their IrConv-LSTM model was tested in 5 cities (Singapore, Chicago, Washington
D.C., New York and London) and performed better than standard benchmarks for
any of them. A recent extensive review on deep learning for bike-sharing prediction
by Jiang noted that the field has witnessed much advancements with especially
the developments of recurrent and convolutional architectures, however due to
most works only evaluate one or two models in isolation it was difficult to compare
techniques [5].
These deep learning models have shown clear superiority over traditional
and machine-learning baselines in capturing temporal and spatial patterns. Yet,
each architecture has its own limitations: RNNs are good at capturing sequential
dependencies at the expense of long training time and difficulty dealing with very
long-range patterns; CNNs advocate computational efficiency and parallelism but
does not have an explicit memory mechanism. These strengths in complementary
guided hybrid systems have driven the design of hybrid structures integrating
multiple neural network components, which will be discussed in the next section.

2.4.4 Hybrid and Attention-Based Approaches
The observation that different architectural features pick up complementary
aspects of time series data has prompted a recent surge in research on hybrid
models, which combine different neural network families inside one forecasting
framework. Zhou et al. (2022) developed a hybrid TCN-GRU model for short-term
bike-sharing demand prediction and tested it with London data from 2015 to 2017
(R-squared 0.984 with MAE and RMSE that is at least 1.98% lower respectively
than standalone LSTM, GRU and ARIMA models as baselines) [42]. They used
the TCN component to capture local temporal pattern by dilated causal
convolutions and a GRU to model dependencies between extracted features, their
results showed that their combined approach improved both components taken
separately.
Beyond these hybrid architectures even more powerful models have been
developed based on the idea that it should be possible to increase the importance
of truly important time steps and input features, rather than treating all positions
in an input sequence identically: attention mechanisms. In the area of bike sharing
and its relation to other transportation aspects, we have seen an increasing
number of paper that demonstrate better-performing models by incorporating
attention into the framework of LSTM [5]. The attention weights also have a certain
interpretability, showing which historical time steps, the model deems most
informative when making a particular prediction, thus providing operational insight
into the temporal demand drivers. Newer work has also looked into channel and
spatial attention modules that act not only in the time dimension across timesteps
but also across feature dimensions, letting the model to focus dynamically in which
input variables are important for each prediction step.

Even with these improvements, the previous hybrid and attention-based
work has several properties that make it less suitable for our particular context of
study. In most studies, one hybrid architecture is suggested and tested, making it
unclear whether the realized benefits are due to this specific combination of
features, or merely due to increased model size. Less is known about purely
theoretical comparisons between more than two types of hybrids when compared
against one or both parents (or vs each other), under standardised experimental
conditions. Moreover, the utilization of a two-stage approach with gradient
boosting and deep-learning feature extraction (e.g., LSTM-XGBoost) creates a
promising direction in bike-sharing domain, which has been less studied but
effective in other transportation forecasting fields. These holes are the impetus for
the systematic, multi-architecture comparison taken on in this thesis.

2.6 Summary and Discussions
This chapter has presented the theoretical background and state-of-the-art
literature in data-driven bike-sharing demand forecasting, including recurrent
neural networks, attentions, feedforward deep neural networks with
static/dynamic structure data input, gradient boosting decision-trees based,
feature engineering-based methods as well as real-world operation scenario of
bike-sharing systems. The related work also discussed how these methods have
been used in statistics based, machine learning based ensembles and mixed deep
learning models for the entire field.
The following review highlights three main voids that are addressed in this
work. The first concerns evaluation methodology. The best known result on the
Seoul Bike Sharing dataset (0.95 R-squared), by Sathishkumar and Cho (2020)
[7], was achieved under a same-time prediction setting, i.e., the model has access

=== PAGE 40 ===
to current hour weather when predicting for the current demand. That's not a real
forecast scenario at all. We employ rigorous one-step-ahead forecast by
forecasting the bike demand at hour t+ 1 using the data until its previous time step
(hour t).
The second gap concerns scope of comparison. Current studies
commonly only evaluate models of one paradigm machine learning studies
compare among GBDT avatars but not deep learning [39], and vice-versa between
different deep learning models, where studies test only two or three architectures
rather than hybrids or boosting counterparts [25] and study about hybrid
combination show just a single configuration that is not compared with other
designs [42]. To our best knowledge, no work makes the systematic comparison
among single models, hybrid architectures, attention-augmented model and
neural-boosting ensemble with exact the same setting.
The third gap consists in under-explored hybrid techniques for bike-sharing
such as mixing channel-spatial attention modules with temporal networks, multi-
scale convolutional processing and recurrent modelling, two-stage deep learning
feature extraction with XGBoost regression model.
We close these gaps with our study that includes 7 architectures: LSTM,
TCN, Hybrid TCN-LSTM, TCN-GRU-Attention, TCN-CBAM-LSTM, LSTM-
XGBoost,and Multi-Scale, TCN+LSTM. We choose those to cover the entire range
between purely individual models and increasingly complex hybrids. They are both
given the same features, they follow the same temporal split pattern and compare
under strictly one-step-ahead conditions so that a direct head to-head comparison
can be done to show which architectural decisions result in meaningful evidence
on how well it plans compared to adding complexity without much return.
